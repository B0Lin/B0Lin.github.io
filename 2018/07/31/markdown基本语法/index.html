Markdown基本语法

2018.07.31 B0Lin  热度 ℃

Markdown简介

Markdown是一个Web上使用的文本到HTML的转换工具，可以通过简单、易读易写的文本格式生成结构化的HTML文档。
HTML 是一种发布的格式，Markdown 是一种书写的格式。Markdown宗旨是实现「易读易写」，并不是HTML的一种替代，它只涵盖了HTML的纯文本编辑部分，因此Markdown的语法很少。不在 Markdown 涵盖范围之内的标签，都可以直接在文档里面用 HTML 撰写，而不需要注明这是HTML的标签。

Markdown基本语法

标题

  1. 使用”=”和”-“分别表示一级标题和二级标题。

This is an H1
=============
This is an H2
-------------

  1. 使用”#”表示标题等级”#”个数对应标题的级数。

# 这是 H1

## 这是 H2

区块引用

在段首使用”>”，表示这是一段区块引用段落。多行区块引用时，不必要每行行首都使用”>”(即使这是可行的)，只需要在该段落第一行行首使用即可，用空行结束多行区块引用方式。如下两种方式都合法：

> This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet,
> consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
> Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

> This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet,
consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

区块引用可以嵌套，根据不同层数使用不同数量的>符号即可，如下：

> This is the first level of quoting.
>
>> This is nested blockquote.

注：如想在区块中添加缩进效果可以添加四个”&nbsp;”。
引用的区块内可以使用其他Markdown语法。
想要在区块中使用>符号，只要在前面加上\符号转义即可。

列表

无序列表使用”*“、”+”或”-“其中一种即可，如下：

* red
+ green
- blue

有序列表使用数字加上英文句号，后面跟空行即可，如下：

  1. bird
  2. McHale

这种写法等同于

<ol>
    <li>bird</li>
    <li>McHale</li>
</ol>

代码区块

要在Markdown中加入代码区块只要缩进四个空格或是一个制表符，Markdown会自动将其转换成<pre>和<code>闭合标签包裹的代码区块，如下：

Here is an example of AppleScript:

tell application “Foo”
    beep
end tell

会被自动转换为：

<p>Here is an example of AppleScript:</p>

<pre><code>tell application “Foo”
    beep
end tell
</code></pre>

一个代码区块会一直持续到没有缩进的那一行。

在代码区块里”&”、”<”和”>”会自动转换成HTML实体，如下：

<div class=”footer”>
    &copy; 2004 Foo Corporation
</div>

会被转换成：

<pre><code><div class=”footer”>
    &amp;copy; 2004 Foo Corporation
    &lt;/div&gt;
</code></pre>

在代码区块中，一般的Markdown语法(例如”*”)不会被转换，这有利于用Markdown语法撰写Markdown语法的文档。

分隔线

使用三个以上的”“、”-“或”_”，来建立分隔线，行内不能有其他内容，”“和”-“之间可以插入空格，如下：

* * *
***
- - -
_ _ _ _ _ _ _ _

链接

Markdown有两种链接形式：行内式和参考式。

  1. 行内式

[content](content_url “title”)

注：”content”为显示在文章中的内容；”content_url”为链接的url；”title”为该”content”在触发onmouseover事件时显示的内容，引号不可省略。

如果要链接到同一主机的资源可以使用相对路径，如下：

[content](relative_path)

注：”relative_path”为”content”的相对路径。

  1. 参考式

[content][id]

注：两个方括号之间也可以添加空格或制表符；”id”要与后文待定义链接的编号对应。”id”可以是数字、英文字母(不区分大小写)、空白或标点符号。

[id]: content_url “title”
[id]: content_url ‘title’
[id]: content_url (title)
注：title可以省略

注：上面三种定义链接的方式都合法。冒号后要接一个空格或制表符；”content_url”与”title”之间要有空格，也可以把”title”属性另起一行书写。

也可以”id”为空时，可以使用如下方法：

[content][]

[content]: content_url “title”
注：title可以省略

强调

使用一对”*”或”_”包裹文字可以实现强调效果，使用单个符号包裹文字将被转换成<em>，使用两个符号则将被转换成<strong>，如下：

*single asterisks*
_single asterisks_
**single asterisks**
__single asterisks__
将分别被转换为：
<em>single asterisks</em>
<em>single asterisks</em>
<strong>double asterisks</strong>
<strong>double asterisks</strong>

强调符号也可以插入在文字内部，如下：

un*frigging*believable

代码

使用”`”(反引号)可以在文章段落中插入一小段代码。

Use the `printf()` function.

如果想在代码中插入反引号，可以单双反引号交替使用(类似于js中字符串中插入引号的方式)，如下：

``There is a literal backtick (`) here.``

在代码区段内(即反引号包裹的范围内)，&apm;和尖括号都会被转换成HTML实体，如下：

Please don’t use any `<blink>` tags.
将被转换为：
<p>Please don’t use any <code><blink></code> tags.</p>

图片

插入图片的方式跟定义链接的方式相同，只是在第一个方括号前面加上感叹号，如下

![img](img_url “title”)
注：title可以省略
或
![img][id]

[id]: img_url “title”
注：title可以省略

Markdown中插入字符：
数学公式符号
转义符号
