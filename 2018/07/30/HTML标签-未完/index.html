简介

HTML(HyperText Markup Language, 超文本标记语言), 他描述并定义了一个网页的基本布局。CSS(Cascading Style Sheets, 层叠样式表), 通常用来描述一个网页的表现与展示效果。 JavaScript通常用来实现网页的功能与行为。

  • “超文本”是指单个网站内或网站之间将网页彼此连接的链接。
  • HTML用”标记”来注明文本、图片和其他内容，以便在Web浏览器中显示。
  • HTML中的标签不区分大小写。

常用的标签

iframe标签

表示嵌套的浏览上下文，有效地将另一个HTML页面嵌入到当前页面中，通常搭配a标签使用。

a标签

全称anchor(锚点)，作用是创建一个超链接。该标签支持全局属性。常见的属性有：

  1. href: 必须属性，该属性值为锚指定一个超文本链接资源。

    该属性有几种取值

    • 无协议绝对地址(href="//google.com"): 使用页面当前协议，同时会发起GET请求。
    • 锚点(href="#element_id"): 实现页面内的跳转，不发起请求。
    • 查询字符串(href="?key=value"): 浏览器会自动把href值作为查询字符串追加到当前url后，同时会发起一个GET请求。
    • 相对路径(href="./index.html"): 会跳转到该页面所在本地路径下的index.html。

    • 伪协议(href="javascript:;"或href="javascript:void(0);"): 点击标签后将执行一段js代码，页面不刷新，不请求。

      跳转到新页面(www.google.com)的几种方式：

    • <a href="http://www.google.com">achor</a>在当前页面中打开。
    • <a href="javascript:void(0);" onclick="window.location.assign('http://www.google.com')"> anchor</a>: 在当前页面打开。
    • <a href="javascript:window.open('http://www.baidu.com');">achor</a>: 在新页面中打开。

    • <a href="javascript:;" onclick="window.open('http://www.baidu.com');">achor</a>: 在新页面中打开。

      注：window.open()函数的第二个可选参数name可以指定target属性(_self、_blank、_parent或_top)或窗口名称。

  2. download: 该属性指示浏览器下载URL而不是导航到URL，因此将提示用户将其保存为本地文件。download属性如果有值，该值将作为下载文件的默认名称。如果不定义download属性，也可以通过在响应头中添加Content-Type: application/octet-steam字段来实现。

  3. target: 该属性指定在何处显示链接资源。

    该属性值有四种默认值：

    • _self: 默认值，在当前页面加载。
    • _blank: 在新窗口中打开。
    • _parent: 在当前上下文(iframe)的父浏览上下文(iframe)中打开
    • _top: 在顶层浏览上下文中打开，如果没有parent框架或浏览上下文，则打开方式同_self。

该属性还通常与iframe的name属性值搭配使用，如下：

<iframe name="ifr" src="#">iframe</iframe>
<a target="ifr" href="ifr_url">anchor</a>
其中a标签的target属性对应iframe的name属性，表示a标签指向的链接将在iframe中打开。

  1. hreflang: 该属性用于指定链接文档的人类语言，仅提供建议，没有内置的功能。

  2. type: 该属性指定链接资源的MIME类型，通常与download属性搭配使用。

  3. rel: 该属性指定了目标对象到链接对象的关系。

form标签

表示了文档中的一个区域，这个区域包含有交互控制元件，用来向web服务器提交信息。常见属性：

  1. action: 提交表单的目的URL。这个值可以被 <button>或者 <input> 元素中的 formaction 属性覆盖。

  2. method: 提交表单的方式，

    • POST: 表单数据会包含在请求正文中。
    • GET: 表单数据会作为查询字符串追加在action属性的值后，一起组成url
      a标签会发起GET请求，而form标签通常用于提交表单，发起POST请求。

  3. name: 在HTML5中，一个文档中如果有多个form表单，则该属性值必须唯一切非空。

  4. target: 表示提交表单后，何处显示收到的回复。用法同a标签。HTML5中这个属性可以被<button>、<input>标签中的属性覆盖。
    可能取值有：

    • _self
    • _blank
    • _parent
    • _top
    • iframe_name

  5. autocomplete: 定义表单中的input元素是否可以被浏览器自动补全。可以被form中的input元素的autocomplete属性覆盖。
    可能取值有：

    • off
    • on: 浏览器可以根据用户之前在form表单中输入值自动补全。

  6. novalidate: 提交表单时是否需要验证，其值为布尔类型。

  7. <span id=”form_enctype>”enctype: 当method属性值为POST时，enctype是提交数据的MIME类型。
    可能取值有：

    • application/x-www-form-urlencoded(默认)
    • multipart/form-data
    • text/plain (HTML5)

input标签

为web表单创建交互式控件。常见属性：

  1. type: 控件类型，
    可以的取值有：

    • button: 按钮，与<button>区别是可否有子元素。
    • checkbox: 复选框。
    • file: 选择要上传的文件。
    • hidden: 不显示在页面上的控件，但它的值会被提交到服务器。
    • image: 图片提交按钮。必须使用 src 属性定义图片的来源及使用 alt 定义替代文本。还可以使用 height 和 width 属性以像素为单位定义图片的大小。
    • password: 隐藏输入值的输入框。
    • radio: 单选按钮。
    • reset: 将表单内容设置为缺省值。
    • text: 单行字段；换行会将自动从输入的值中移除。

    • submit: 表单提交按钮

      下面是HTML5中新增的控件类型：

    • time: 时分秒控件。
    • url: 用于编辑URL的字段。
    • week: 星期-年控件。
    • color: 用于指定颜色的控件。
    • date: 年月日控件。
    • datetime: 基于 UTC 时区的日期时间输入控件。
    • datetime-local: 用于输入日期时间控件，不包含时区。
    • email
    • month: 输入年-月。
    • number: 输入浮点数。
    • range: 用于输入不精确值控件。
    • search:

    • tel:

      注：
      <input type="submit" value="提交">
      <button>提交</button>
      <input type="button">
      <button type="button">
      以上四种按钮唯一存在于表单中，只有前两种可以作为提交按钮

  2. autocomplete: 浏览器是否自动补全内容，根据不同的type，浏览器会有不同的策略。

  3. autofocus: 页面加载后自动聚焦该属性为true的控件，属性在页面中唯一。

  4. name: 拥有name属性的控件可以一起作为表单数据提交。

  5. required: 必填。

  6. placeholder: 占位值

  7. value: 初始值。

    注:
    placeholder是不可删除的，value是可以删除的。

button标签

按钮默认样式跟宿主环境有关，可以使用CSS更改。常用属性：

  1. disabled: 不可选中。
  2. name和value: 跟表单其他数据一起以name=value键值对的形式提交。
  1. type: 定义<button>类型，可选属性值有：
  • submit: 表单中按钮唯一时的默认属性值。
  • reset: 重置所有组件为初始值。
  • button
  • menu: 打开一个<menu>元素定义的菜单(实验中的功能)。

下面是HTML5新增属性

  1. autofocus: 布尔属性，页面加载后自动获得焦点。

  2. form: 属性值为同一文档中的某一个<form>元素的id属性，该<button>将与这个<form>元素相关联，使得<button>可以放在文档中的任意位置。

  3. formenctype: 指定表单提交内容的类型。如果定义了该属性，且<button>的type属性为submit，则将覆盖<form>中的enctype值。

  4. formaction、formmethod、formnovalidate和formtarget的用法同formenctype，都会覆盖<form>元素中对应的属性。

table标签

给HTML文档添加二维表格数据。常用属性：

  1. align: 定义表单对其方式。可选属性为：left、center、right。CSS中的替代方式为：设置 margin-left 、margin-right 为 auto ( 或者 margin 设置为 0 auto) 来实现类似于 align 方式的效果。

  2. bgcolor: 定义表单颜色，可以使用预先定义的代表颜色的字符串(e.g. black)，或者使用颜色对应的RGB。CSS中的替代方式为： CSS 中的 background-color 属性能够和bgcolor属性达到一样的效果。

  3. border: 定义表格边框像素宽度。CSS中的替代方式为：CSS 属性 border, border-color, border-width 和 border-style可实现相同效果。

  4. cellpadding: 定义了表格单元的内容和边框之间的空间。CSS中的替代方式为： 在<table> 元素上使用 CSS属性值为 collapse 的 border-collapse 属性，在 <td>元素上使用属性 padding，以达到类似于 cellpadding 的效果。

  5. cellspacing: 定义了两个单元格之间空间的大小。CSS中的替代方式为：在 <table> 元素上使用 CSS 的属性 border-spacing ，以达到类似于 cellspacing 的效果。

  6. frame: 这个枚举属性定义了包围在表格周围的框架的哪个边必须显示。CSS中的替代方式为：在 <table> 元素上使用 CSS 的属性 border-style 和 border-width ，以达到类似于 frame 的效果。

  7. rules: 这个枚举属性定义了在一个表格中规则的显示位置。CSS中的替代方式为：使用 border 属性，在适当的 <thead>, <tbody>, <tfoot>, <col>或 <colgroup>元素上。

  8. summary: 该属性定义了一个替代文本，当表格无法在用户代理中显示的时候用来描述表格。

  9. width: 该属性定义了表格的宽度。CSS中的替代方式为：使用 CSS定义和设计样式。使用 width 属性代替。

注：尽量使用CSS样式来定义表单样式，以上<table>属性都已经废弃

常用在table标签内的子元素有:

  • thead
  • tbody: 并非是<table>中必须的标签，且同一表单元素中可以存在多个。
  • tfoot

注：即使以上三个标签没有按照thead->tbody->tfoot的顺序定义，浏览器也会按照此顺序解析。

  • colgroup : 表格列组标签用来定义表格中的一组列表。

使用方法如下：

<colgroup>
<col span="2" style="background-color:red">
<col style="background-color:yellow">
</colgroup>
